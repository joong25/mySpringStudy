DI 의존객체 전달방식 

- 생성자 방식
생성자를 통해 생성될 때 파라미터로 넘겨받아 의존성을 주입하는 방식

장점 
빈 객체를 생성하는 시점에 모든 의존 객체가 주입
ex) 세터 메서드 방식은 세터 메서드를 사용해서 필요한 의존 객체를 전달하지 않아도
    빈 객체가 생성되기 때문에 객체를 사용하는 시점에 NullPointerException 발생 가능
- 프로퍼티 방식(세터 메서드 방식)
set메소드를 이용하여 의존성을 주입해주는 방식
setParameter()라는 set 메소드가 있다면 실제로 bean을 등록할 때는
set을 뺀 Parameter()에서 맨앞자리를 소문자로 바꿔준 parameter로 사용

메서드 이름이 set으로 시작
set 뒤에 첫 글자는 대문자로 시작
파라미터 1개
리턴 타입 void

장점
세터 메서드 이름을 통해 어떤 의존 객체가 주입되는지 알 수 있다.
ex) 생성자 파라미터 개수가 많을 경우 각 인자가 어떤 의존 객체를 설정하는지
알아내야 할 때 설정 메서드 이름만으로 어떤 의존 객체 설정하는 지 유추 가능
